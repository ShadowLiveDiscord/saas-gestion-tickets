04 — Base de données
IndexedDB — La vraie base de données du navigateur

Remplace localStorage : transactions ACID, index, capacité quasi illimitée, API asynchrone.

⚖️ localStorage vs IndexedDB
CritèrelocalStorageIndexedDB
Capacité~5 MoQuasi illimitée
StructureClé-valeurObject Stores + Index
APISynchrone (bloquant)Async (Promise)
RequêtesScan completIndex rapides
Transactions✕ Non✓ ACID
TypesString seulementObjets JS natifs
🗄️ Object Stores (≈ tables SQL)
🎫
tickets
Clé : id — Index : userEmail, status, priority, date
👤
users
Clé : id — Index : email (unique), role
🎧
agents
Clé : id — Index : email (unique)
⚡ API asynchrone — Exemples de requêtes
// Ouvrir la BDD (crée les tables si besoin) const db = await openDB('ticketflow_db', 1); // SELECT * FROM tickets WHERE userEmail = ? const myTickets = await TicketDB.getByUser('user@mail.fr'); // INSERT OR REPLACE INTO tickets VALUES (?) await TicketDB.save({ id: Date.now(), title, status: 'ouvert' }); // DELETE FROM tickets WHERE id = ? await TicketDB.delete(ticketId); // Transactions ACID — plusieurs écritures atomiques await TicketDB.saveMany([ ticket1, ticket2, ticket3 ]);

Parcours utilisateur
1
Connexion sécurisée
L'utilisateur entre son email et mot de passe. La session est stockée et il est redirigé automatiquement vers son dashboard.
2
Tableau de bord personnel
Il voit uniquement ses propres tickets chargés depuis IndexedDB, avec des statistiques en temps réel (total, en cours, résolus, critiques).
3
Créer un ticket
Il remplit un formulaire : titre, catégorie, priorité, description. Le ticket est enregistré dans IndexedDB et apparaît instantanément.
4
Fil de discussion
En cliquant sur "Voir", il accède au détail du ticket et peut échanger des messages avec l'agent en temps réel.
5
Recherche & filtres
Il peut filtrer ses tickets par statut (ouvert, en cours, résolu) et par priorité, ou faire une recherche libre par mot-clé.

📖 Glossaire des termes techniques
Passez la souris sur les termes soulignés en pointillé dans les slides pour voir leur définition. Appuyez sur G pour ouvrir/fermer.
IndexedDB BDD
Base de données intégrée aux navigateurs modernes. Contrairement à localStorage, elle supporte les transactions, les index et peut stocker de grandes quantités de données de manière structurée.
localStorage Stockage
Système de stockage simple du navigateur. Stocke des paires clé-valeur en texte uniquement, limité à ~5 Mo. Pratique pour des petites données mais sans requêtes avancées.
ACID BDD
Propriétés garantissant la fiabilité des transactions : Atomicité (tout ou rien), Cohérence (état valide), Isolation (transactions indépendantes), Durabilité (données persistées).
async / await JS
Syntaxe JavaScript moderne pour gérer des opérations asynchrones (réseau, BDD...) de façon lisible, sans bloquer l'interface utilisateur pendant l'attente du résultat.
Feature Flag Archi
Interrupteur logiciel permettant d'activer ou désactiver une fonctionnalité en production sans redéployer le code. Utile pour les démos, les tests A/B ou les lancements progressifs.
Object Store IndexedDB
Équivalent d'une table SQL dans IndexedDB. Stocke des objets JavaScript natifs (pas de sérialisation nécessaire) identifiés par une clé primaire (keyPath).
SaaS Modèle
Software as a Service : logiciel accessible via un navigateur web, sans installation, généralement proposé par abonnement mensuel. Ex : Slack, Notion, Jira.
DOM Web
Document Object Model : représentation de la page HTML sous forme d'arbre d'objets manipulables par JavaScript. document.getElementById() permet d'accéder aux éléments.
CDN Réseau
Content Delivery Network : réseau de serveurs distribués géographiquement servant des fichiers (CSS, JS...) rapidement. Utilisé ici pour charger jsPDF sans installation locale.
CRUD BDD
Create, Read, Update, Delete : les 4 opérations de base sur une base de données. Toute l'API TicketDB (save, getAll, getByUser, delete) implémente ces opérations sur IndexedDB.
Session Auth
Données identifiant un utilisateur connecté. Dans TicketFlow, la session est stockée dans localStorage (clé tf_user) et lue à chaque chargement de page pour rediriger l'utilisateur.
ES6+ JS
Versions modernes de JavaScript (depuis 2015). Introduit les classes, modules, arrow functions, destructuration, async/await, et bien d'autres fonctionnalités utilisées dans TicketFlow.
