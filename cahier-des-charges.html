Présentation du projet

Contexte

TicketFlow est une application web de type SaaS (Software as a Service) permettant à des organisations de gérer leurs demandes de support client sous forme de tickets. Le projet est développé dans le cadre d'un cours, en HTML, CSS et JavaScript pur, sans framework ni backend.

Problème résolu

Les équipes de support manquent souvent d'un outil centralisé pour recevoir, trier et traiter les demandes clients. TicketFlow propose une solution simple, accessible depuis n'importe quel navigateur, sans installation.

Public cible

ProfilBesoin principal
Petites et moyennes entreprisesCentraliser les demandes clients sans outil complexe
Équipes de supportTraiter et suivre les tickets efficacement
Clients / utilisateurs finauxSoumettre une demande et suivre son avancement

Un ticket passe par plusieurs statuts depuis sa création jusqu'à sa résolution.

Sécurité et contrôle d'accès

Mécanismes de protection

MécanismeDescription
Vérification de sessionChaque dashboard vérifie la présence de tf_user au chargement
Vérification du rôleRedirection automatique si le rôle ne correspond pas à la page
Feature flagsRedirection vers "Prochainement" si le module est désactivé
Protection adminLe panel admin ne peut pas être désactivé par les feature flags
Note : cette version utilise une authentification simulée côté client (localStorage). Dans une version de production, l'authentification devra être gérée par un backend sécurisé avec tokens JWT ou sessions serveur.

TermeDéfinition
SaaSSoftware as a Service — logiciel accessible via un navigateur web, sans installation
TicketDemande de support créée par un utilisateur, traitée par un agent
Feature FlagInterrupteur permettant d'activer ou désactiver une fonctionnalité sans modifier le code
DashboardTableau de bord regroupant les informations et actions principales d'un rôle
localStorageStockage de données dans le navigateur, persistant entre les sessions
